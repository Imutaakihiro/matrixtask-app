# アイゼンハワーマトリクスTodoアプリ - 壁打ちメモ

**作成日**: 2026-02-11

## コンセプト

アイゼンハワーマトリクスを使った個人向けWebアプリ。「分類は素晴らしいけど続かない」問題を解決する。

## ターゲット

- 個人ユーザー
- Webアプリ（ブラウザベース）

## 技術スタック

- TypeScript
- Node.js v24.11.0
- （既存プロジェクトのスタックを使用）

## コア機能（MVP）

### 必須機能
- ✅ 受信箱（まず放り込む場所）
- ✅ マトリクス（4象限で分類）
  - 重要×緊急
  - 重要×緊急でない
  - 重要でない×緊急
  - 重要でない×緊急でない
- ✅ 今日やること（実際の作業ビュー）
- ✅ タスクのCRUD操作
- ✅ ドラッグ&ドロップ
  - 象限間の移動
  - 受信箱からマトリクスへ
  - マトリクスから「今日」へ
- ✅ ショートカット（例: `Ctrl+K`で素早く追加）
- ✅ 自然言語入力（例: 「明日までにレポート #重要」）

## UI/UX方針

### デザイン
- **Notionライクなシンプル白UI**
- クリーン・ミニマル
- 視覚的にわかりやすい

### レイアウト構成

```
┌─────────────────────────────────────────┬──────────────┐
│         受信箱（インライン表示）          │              │
│  ┌────────────────────────────┐         │              │
│  │ + タスクを追加...           │         │              │
│  │ ・未分類タスク1             │         │   サイド     │
│  │ ・未分類タスク2             │         │              │
│  └────────────────────────────┘         │  今日やること │
├─────────────────────────────────────────┤   ┌────┐    │
│                                         │   │タスク│    │
│            マトリクス                   │   │タスク│    │
│        （メイン・常時表示）              │   │タスク│    │
│                                         │   └────┘    │
│  ┌──────────┬──────────┐              │              │
│  │重要×緊急  │重要×緊急× │              │              │
│  │          │          │              │              │
│  ├──────────┼──────────┤              │              │
│  │緊急      │          │              │              │
│  │          │          │              │              │
│  └──────────┴──────────┘              │              │
└─────────────────────────────────────────┴──────────────┘
```

### 画面配置の決定事項
- **受信箱**: マトリクスの上部にインライン表示
- **マトリクス**: 常時表示（メインエリア）
- **今日やること**: 右サイドに固定表示

## 使いやすさのポイント

### 1. 分類の難しさ問題を解決
- まず受信箱に放り込むだけでOK
- 後でまとめて分類
- 自然言語入力でラベル付け可能

### 2. 実際の行動につなげる
- 「今日やること」ビューで実行にフォーカス
- マトリクスから「今日」へドラッグでピン留め
- 完了したらアーカイブ

### 3. 入力の手軽さ
- キーボードショートカット
- 自然言語でパース（期限、重要度など）

## 検討事項

### 自然言語入力の解釈
例: 「明日までにレポート作成 重要」

- 「明日まで」→ 期限設定 → 緊急判定？
- 「重要」→ ユーザー明示
- → まず受信箱に入れる？それとも直接マトリクス？

**要検討**: どこまで自動判定 vs ユーザー指定するか

### 「今日やること」への流れ
- 毎朝、重要×緊急タスクを自動で「今日」に追加？
- ユーザーが手動でピン留め？
- 両方可能にする？

## 設計思想

**「さっと書いて、後で整理」**

受信箱 → マトリクス → 今日やること、という自然な流れを作る。
無理に最初から完璧に分類させない。行動につなげることを重視。
